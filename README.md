# Scholarmart Campus Marketplace MVP

A trusted, mobile-first campus marketplace platform designed for Nigerian university students, where students can, buy,sell and connect.

---

## Technical Stack
- **Frontend**: HTML5, Vanilla CSS3, Vanilla JS (ES2025+), Web App Manifest, Service Workers (PWA)
- **Backend**: Node.js, Express.js
- **Database**: PostgreSQL (standard production SQL queries with an automated JSON sandbox file database fallback)
- **Authentication**: JSON Web Tokens (JWT) + bcryptjs hashing
- **File Uploads**: Multer
- **Payments**: Paystack API with Flat Splits

## Quick Start (Sandbox / JSON Fallback Mode)

Scholarmart is configured to run out-of-the-box in **Dual-Mode**. If no PostgreSQL server is running, the server logs a warning and automatically falls back to an internal JSON-based file store (`backend/data/db_store.json`), allowing you to test login, registration, catalog additions, checkouts, and admin analytics without any database installations.

### 1. Install Dependencies
Navigate to the `backend` folder and install packages:
```bash
cd backend
npm install
```

### 2. Configure Environment variables (Optional)
Create a `.env` file inside the `backend` directory to override defaults:
```env
PORT=3000
JWT_SECRET=your-custom-jwt-secret-key
APP_URL=http://localhost:3000

# PAYSTACK SECRET KEY
# (If missing, Scholarmart runs sandbox payment gateways and returns mock subaccounts)
PAYSTACK_SECRET_KEY=

# POSTGRES DATABASE DETAILS
# (If unreachable, Scholarmart falls back to local JSON database)
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=postgres
DB_NAME=scholarmart
```

### 3. Run Automated E2E Tests
To run the automated integration tests checking registration, login, OTP approvals, product listing filters, checkouts, and admin splits reports:
```bash
npm run test
# or
node test-api.js
```

### 4. Start the Application
Start the local Express server:
```bash
npm start
```
Open **[http://localhost:3000](http://localhost:3000)** in your browser to experience the mobile marketplace app.

## Manual Verification Guide

### 1. Default Admin Log-In
- Click the **Profile** tab in the sticky bottom navigation.
- Sign in using the default admin account:
  - **Email**: `admin@scholarmats.com`
  - **Password**: `AdminPassword098`
- Review verification logs, reported product cards, list users, reset passwords, and track platfee collections.

### 2. Student Signup & Email OTP Check
- Register as a student using an email ending in `.edu.ng` (e.g. `chinedu@coou.edu.ng`).
- Go to your dashboard and click **Send OTP to Student Email**.
- The OTP is logged to the Express terminal console. Enter this code to auto-approve your student badge.

### 3. ID Card Verification
- If signing up with a non-student email, upload a picture in the dashboard.
- Log in as the Admin and click **Approve** in the **Verifications** tab to verify their account.

### 4. Products listing & WhatsApp direct chat
- Click the green **`+` FAB button** to add listings (Mattress, Gadgets, Books, etc.).
- Browse items in the **Marketplace** catalog, search names, apply campus filters, and click **WhatsApp Seller** to automatically open WhatsApp carrying a prefilled interest text template.

## Production Deployment Guide (PostgreSQL & Live Paystack)

When deploying to a live staging or production environment (e.g., Render, Heroku, AWS, or Vercel), execute the following steps to link PostgreSQL and active Paystack split payments:

### 1. Set Up PostgreSQL
1. Create a PostgreSQL instance (e.g. Supabase, RDS, ElephantSQL).
2. Connect to the instance and execute the DDL commands inside `backend/schema.sql` to initialize tables and pilot campus list lookups:
   ```bash
   psql -h <your-host> -U <your-user> -d <your-db-name> -f backend/schema.sql
   ```

### 2. Set Up Paystack Subaccounts
1. Log in to your [Paystack Dashboard](https://dashboard.paystack.com/) and go to **Settings > API Keys** to retrieve your **Secret Key**.
2. Save this key in your production environment variables:
   ```env
   PAYSTACK_SECRET_KEY=sk_live_xxxxxxxxxxxxxxxxxxxxxx
   ```
3. During vendor registration, the Express backend automatically invokes the Paystack Subaccount API (`POST https://api.paystack.co/subaccount`) using the vendor's bank name and account details, storing the return code (`ACCT_xxxx`) for future flat split transfers.

### 3. Deploy Server
Deploy the codebase to a Node hosting provider. Make sure you map the correct environment variables:
```env
PORT=80
DB_HOST=your-postgres-host.com
DB_PORT=5432
DB_USER=your-database-user
DB_PASSWORD=your-database-password
DB_NAME=your-database-name
JWT_SECRET=a-secure-random-string-like-uuid-for-json-tokens
APP_URL=https://your-production-app.herokuapp.com
PAYSTACK_SECRET_KEY=sk_live_xxxxxxxxxxxxxxxxxxxxxx
```
The application will automatically detect the database and run in Postgres mode, processing real transaction charges and webhook callbacks.
